### MUST-FIX (blocks demo or real bug)

#### M1 — `perDefenderBanditLootShare` returns 0 in normal gameplay
**File:** `packages/contracts/src/diamond/lib/LibBanditCombat.sol:575-578`

```solidity
function perDefenderBanditLootShare(uint256 loot, uint256 nDefenders) public pure returns (uint256) {
    if (nDefenders == 1) return loot;
    return ((loot / RESOURCE_UNIT) / nDefenders) * RESOURCE_UNIT;
}
```

`RESOURCE_UNIT = 1e18`. The two-step truncation collapses to zero whenever `loot < nDefenders * 1e18`. The call site is `distributeBanditLootToDefendingClans` at line 367-371, fed by `banditLootDrop(carry) = carry * 5000 / 10000` (50%).

**Realistic trigger:** Clan vault wood = 5e18 (below starting 20e18, easily reached after raids). Bandit steals 5e18 × 20% = 1e18. On defeat, drop = 1e18 × 50% = 5e17. With ≥2 defenders: `((5e17 / 1e18) / 2) × 1e18 = 0`. Defenders get nothing; resources are permanently lost to the bandit-defeat sink.

No test covers fractional-loot defender distribution (`test/BanditAttackResolution.t.sol` only uses 8e18 / 100e18 carry values, always perfect multiples of 1e18).

**Fix:** Reorder to divide-then-truncate: `(loot / nDefenders / RESOURCE_UNIT) * RESOURCE_UNIT`. Add a thin happy-path test with `loot < nDefenders * RESOURCE_UNIT` to prevent regression.

#### M2 — ABI artifact is stale vs source; CI doesn't catch it
**Files:** `packages/contracts/abi/IClanWorld.json` (last modified 2026-05-10 14:54 -0400), `packages/contracts/src/IClanWorld.sol` (last modified 2026-05-10 20:02 -0400), `.github/workflows/chainclient-abi.yml`

The ABI artifact checked into the repo is **~5 hours behind** the Solidity source. The chainclient-abi workflow runs `pnpm test:chainclient-abi`, which validates that `IChainClient.ts`'s embedded `CLAN_WORLD_ABI` matches the JSON artifact, but it never re-runs `forge build` to detect drift between Solidity source and the JSON artifact. So if a contributor edits `IClanWorld.sol` and forgets to commit the rebuilt artifact, CI passes and the indexer + frontend silently work against an outdated ABI.

The current 5-hour gap is benign (no schema-breaking change in that window per `git log`), but the gate is structurally broken. **Fix:** add a `forge build && git diff --exit-code packages/contracts/abi/` step to the workflow, or auto-regenerate the artifact on PR.

#### Backend (Convex)

- **S7 — `agentLogs.postLog` is unguarded public mutation:** `apps/server/convex/agentLogs.ts:10`. Every other state-mutating Convex mutation (`comms.seed*`, `inft.mirror*`, `bulletins.seedBulletin`) uses `requireIndexerSecret`. `postLog` is callable by any anonymous client and lets them write to the cockpit log feed. Add the same guard or make it `internalMutation`.
- **S8 — `seedVaultMovement` hardcodes `logIndex: 0` with weak txHash:** `apps/server/convex/vault.ts:317-343`. Synthetic txHash is `seed-${Date.now()}-${Math.random().toString(36).slice(2,8)}` (32 bits of entropy on the suffix) and the logIndex is always 0. If called twice in the same millisecond, the dedup index `by_tx_log` can collide. Use a UUID or per-event sequence number.
- **S9 — `comms.getCombinedComms` does full-table scans for "received by me":** `apps/server/convex/comms.ts:29,46`. Capped at `cap * 2` so not catastrophic, but no index supports `toClanIds`/`targetClanId` lookups. Add an index or denormalize a reverse pointer.
- **S10 — 7 of 8 `v.any()` validators in `schema.ts` lack justifying comments:** lines 41, 120, 121, 122, 138, 139 (and one more). The intent is documented at line 70 for `chainEvents.args` but not propagated. Add `// intentional: dynamic struct, see comment at line 70` to each.
